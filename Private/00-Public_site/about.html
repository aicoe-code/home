Purpose

The AICOE site helps teams understand, design, build, deploy, and operate AI solutions safely and effectively. It curates opinionated guidance, reusable patterns, and reference assets that reduce time‑to‑value while promoting good engineering and governance.

Audience

  • Practitioners: data/ML/software/platform engineers
  • Decision makers: product, delivery, and technical leads
  • Enablers: security, risk/compliance, architecture boards

How This Site Publishes

  1. Content is authored in Private/00-Public_site/ (tracked in Git).
  2. Drafts use the x_ prefix and are excluded from publishing.
  3. On push to main, a workflow syncs to docs/, fingerprints CSS/JS filenames by content hash, updates HTML references, and commits the result.
  4. GitHub Pages serves the site from docs/.

Tip: In source HTML, reference assets by their plain names (e.g., styles.css); the workflow rewrites them to hashed names during publish.

Writing Guidelines

  • Be direct and practical; show the minimum that works, then extensions.
  • Use semantic headings, alt text for images, and accessible color contrast.
  • Each guide should include prerequisites, exact steps, validation, and cleanup.
  • Keep secrets out of the repo; reference environment variables or vaults instead.

Contribute

Open a PR or push small, focused commits to main. For substantive guidance, request a second reviewer (content + technical). Prefer canonical sources and note version/date where relevant.
